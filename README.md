# Gantt Desk

外部AIで整形したJSONを取り込み、確認・編集・可視化するためのローカル保存型ガントチャートです。

## 特徴

- 初回起動時は、入力不要のサンプル進行表を表示（編集を始めると、そのまま作業データとして保存）
- ブラウザ内のローカル保存（`localStorage`）
- AI機能・APIキー・サーバー処理なし
- JSONの貼り付け、ファイル選択、ドラッグ＆ドロップ
- 反映前のJSON検証とプレビュー
- 「追加」と「置き換え」を選択可能
- JSON完全バックアップとAI連携用JSONの書き出し
- 土日を連続帯で表示し、タスク色を主役にする静かなガント表示
- 日本の基本祝日と、JSONで追加した独自休業日を薄い赤背景で表示
- タスクバーのドラッグ移動、右端ドラッグによる終了日変更
- 右サイドの編集パネル、Undo / Redo、端末内自動保存

## 表示設定

上部ツールバーの **表示設定** から、進行表の見え方を調整できます。

- 表示密度：ゆったり／標準／最小
- 行の高さ：28〜72px のつまみで調整
- 日付幅：10〜56px のつまみで調整
- 左の一覧幅：つまみで調整
- カテゴリ表示：表示／色だけ／非表示
- 期間をタスクに合わせる
- 最小表示：ワンクリックで高密度表示へ切り替え

共有前やスクリーンショット前は、`最小表示` と `カテゴリ：色だけ` を使うと一覧性が上がります。

## カレンダー表示

ヘッダー下に、常時見える凡例を表示します。

- 通常日：白
- 土日：薄いニュートラルグレー背景
- 祝日・休業日：薄い赤背景
- 今日：濃紺の縦線のみ

土曜・日曜の文字色分けは行わず、曜日文字はグレーで統一します。スマホ幅では凡例を短縮表示し、さらに狭い画面では色・線アイコンだけにします。

## 保存・管理

右上の **保存** から、ブラウザ内に保存したガントを管理できます。

- 新しいガントを作成
- 現在のガントを複製
- 保存済みガントの切り替え
- 不要なガントの削除
- 既存の1件保存データは、初回起動時に自動でプロジェクト一覧へ移行
- モバイルでは `…` メニューから利用可能

データはこのブラウザと端末にだけ保存されます。別の端末や別ブラウザへの同期は行いません。

## カテゴリ一括変更

**保存** の `カテゴリ一括変更` タブから、カテゴリをまとめて整えられます。

- カテゴリ追加
- カテゴリ名の変更（紐づく全タスクへ反映）
- カテゴリ削除（移動先カテゴリを指定して全タスクを移動）
- 複数タスクを選択して、カテゴリを一括変更

## 共有・出力

右上の **共有・出力** から、進行表を説明資料として整えて出せます。

- 共有ビュー：編集ボタンを隠し、案件名・対象期間・最終更新日時・表示件数・凡例を表示
- 共有ビューでは、自動で高密度表示に寄せて一覧性を高める
- 共有ビューのヘッダー上から、PNG、Excel、画像コピー、共有設定、編集に戻る操作ができます
- カレンダー凡例とカテゴリ凡例を分けて表示します
- 共有範囲：全体／今月／直近4週間／今見えている範囲／指定期間
- PNG：16:9、A4横、縦長
- PNG／Excelでも、画面表示と同じカレンダー色ルールを使う
- 画像のクリップボードコピー：Slack、Teams、PowerPointへの貼り付け向け
- 共有用Excel：`共有用ガント` と `データ一覧` の2シートを含む `.xlsx`

出力時は、案件名・表示期間・最終更新日・凡例・メモ・完了タスクを含めるか選べます。

## GitHub Pagesでの公開

1. このリポジトリの **Settings** → **Pages** を開く
2. **Build and deployment** の Source を **GitHub Actions** に設定
3. `main` ブランチへpushすると、`.github/workflows/deploy-pages.yml` が公開を実行します

公開後のURLは通常、`https://silovar-uk.github.io/gantt/` です。

## JSONの基本形式

```json
{
  "version": 1,
  "title": "開幕プロモーション",
  "memo": "全体共有事項",
  "view": {
    "start": "2026-07-01",
    "end": "2026-09-30"
  },
  "holidays": [
    { "date": "2026-08-13", "name": "社内休業日" }
  ],
  "tasks": [
    {
      "name": "KV初稿提出",
      "start": "2026-07-08",
      "end": "2026-07-10",
      "category": "制作",
      "color": "blue",
      "milestone": false,
      "deadline": false,
      "note": "確認者：広報・営業"
    }
  ]
}
```

### holidays の書き方

日本の基本祝日はアプリ側で表示します。案件固有の休業日は、JSONの `holidays` に追加できます。

```json
"holidays": [
  "2026-08-13",
  { "date": "2026-08-14", "name": "夏季休業" }
]
```

### 利用できる色

`gray` / `blue` / `green` / `amber` / `red` / `purple`

### 読み込み時の扱い

- `end` がない場合は、`start` と同じ日付を指定してください
- `deadline: true` は `milestone: true` のときだけ有効です
- 未登録カテゴリは自動で追加されます
- 未対応の色は `gray` に補正されます
- 日付不正や終了日が開始日より前のタスクは、反映前にエラーとして表示されます
- `holidays` は日本の基本祝日に追加する休業日として扱います

## データについて

日常の編集中データは、このブラウザの `localStorage` に保存されます。ブラウザデータを削除すると消えるため、節目ごとに「完全バックアップJSON」を書き出してください。
